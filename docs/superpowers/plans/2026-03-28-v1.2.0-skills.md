## Authentication

Set `REPLICATE_API_TOKEN` in your environment:

```bash
source .env
```

All requests require:

```
Authorization: Bearer $REPLICATE_API_TOKEN
Content-Type: application/json
```

## Workflow

1. **Choose the right model** — search via API or consult `references/` for preferred models
2. **Fetch model schema** — GET `/v1/models/{owner}/{model}` to confirm input parameters
3. **Create prediction** — POST to `/v1/models/{owner}/{model}/predictions`
4. **Poll for result** — GET prediction URL until `status` is `"succeeded"`
5. **Return output** — usually URLs to generated files; download and back up within 1 hour

## Model Discovery

Search:

```bash
curl -s "https://api.replicate.com/v1/models?query=video+generation" \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" | jq '.results[].name'
```

Browse collections (curated by Replicate staff):

```bash
curl -s "https://api.replicate.com/v1/collections" \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" | jq '.[].slug'
```

Use official models — they are always running, have stable schemas, and predictable pricing.

## Running Models

**Use `/v1/models/{owner}/{model}/predictions`** — works for both official and community models.

**Fetch the schema before calling a model:**

```bash
curl -s "https://api.replicate.com/v1/models/{owner}/{model}" \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" | jq '.latest_version.openapi_schema.components.schemas.Input'
```

**Create a prediction (async):**

```bash
curl -s -X POST "https://api.replicate.com/v1/models/{owner}/{model}/predictions" \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"input": {"prompt": "..."}}' | jq '{id, status, urls}'
```

**Poll until complete:**

```bash
curl -s "https://api.replicate.com/v1/predictions/{id}" \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" | jq '{status, output, error}'
```

**Synchronous (fast models only):**

Add `Prefer: wait` header — blocks until the prediction completes. Only use for models that finish in seconds.

## Guidelines

- Always validate inputs against the model schema before submitting
- Omit optional parameters unless you have a reason to set them
- Use HTTPS URLs for file inputs, not local paths
- Fire concurrent predictions — don't wait for one before starting the next
- Output URLs expire after 1 hour — download and back them up immediately
- Use webhooks for long-running predictions in production

# Seedance 1.5 Pro

**Model ID:** `bytedance/seedance-1-5-pro`
**API endpoint:** `https://api.replicate.com/v1/models/bytedance/seedance-1-5-pro/predictions`
**Use:** Image-to-video and text-to-video generation

## Key Inputs

| Parameter | Type | Default | Notes |
|---|---|---|---|
| `prompt` | string | required | Motion description — see `writing-seedance15-prompt` skill |
| `duration` | integer | 5 | Clip length in seconds (2–12) |
| `aspect_ratio` | string | `"16:9"` | `16:9` \| `9:16` \| `4:3` \| `3:4` \| `1:1` \| `21:9` \| `9:21` |
| `resolution` | string | `"720p"` | `480p` \| `720p` \| `1080p` |
| `fps` | integer | 24 | Frames per second (24 only) |
| `image` | string | — | HTTPS URL or data URI for image-to-video |
| `generate_audio` | boolean | false | Enable synchronized audio |

## Curl — Text-to-Video

```bash
source .env
PREDICTION=$(curl -s -X POST \
  "https://api.replicate.com/v1/models/bytedance/seedance-1-5-pro/predictions" \
  -H "Authorization: Bearer $REPLICATE_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "input": {
      "prompt": "A horse gallops across a dusty road, dust rising behind its hooves, camera tracking side pan",
      "duration": 5,
      "aspect_ratio": "16:9",
      "resolution": "720p"
    }
  }')
echo $PREDICTION | jq '{id, status}'
PREDICTION_URL=$(echo $PREDICTION | jq -r '.urls.get')

# Poll until complete
while true; do
  STATUS=$(curl -s "$PREDICTION_URL" -H "Authorization: Bearer $REPLICATE_API_TOKEN")
  echo $STATUS | jq '{status}'
  if echo $STATUS | jq -e '.status == "succeeded" or .status == "failed"' > /dev/null; then
    echo $STATUS | jq '{status, output, error}'
    break
  fi
  sleep 2
done
```

## JavaScript

```js
const response = await fetch(
  "https://api.replicate.com/v1/models/bytedance/seedance-1-5-pro/predictions",
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.REPLICATE_API_TOKEN}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      input: {
        prompt: "A horse gallops across a dusty road, camera tracking side pan",
        duration: 5,
        aspect_ratio: "16:9",
        resolution: "720p",
      },
    }),
  }
);
const prediction = await response.json();

// Poll
let result = prediction;
while (result.status !== "succeeded" && result.status !== "failed") {
  await new Promise((r) => setTimeout(r, 2000));
  const poll = await fetch(result.urls.get, {
    headers: { Authorization: `Bearer ${process.env.REPLICATE_API_TOKEN}` },
  });
  result = await poll.json();
}
console.log(result.output); // array of video URLs
```

---
name: using-falai-model
description: Call AI models via fal.ai's API — submit to queue, poll status, retrieve results, and handle webhooks.
---

## Authentication

Set `FAL_KEY` in your environment:

```bash
source .env
```

All requests require:

```
Authorization: Key $FAL_KEY
Content-Type: application/json
```

## Calling Modes

Choose based on your use case:

| Mode | When to use | How |
|---|---|---|
| **Sync** | Fast models (<10s) | POST to `https://fal.run/{model-id}` |
| **Queue (recommended)** | Most models | POST to `https://queue.fal.run/{model-id}` → poll → fetch result |
| **Webhook** | Production / long jobs | Add `?fal_webhook=https://...` to queue submit |

## Queue Workflow (Recommended)

**Step 1 — Submit:**

```bash
curl -s -X POST "https://queue.fal.run/{model-id}" \
  -H "Authorization: Key $FAL_KEY" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "..."}' | jq '{request_id, status}'
```

**Step 2 — Poll status:**

```bash
curl -s "https://queue.fal.run/{model-id}/requests/{request_id}/status?logs=1" \
  -H "Authorization: Key $FAL_KEY" | jq '{status, queue_position}'
```

States: `IN_QUEUE` → `IN_PROGRESS` → `COMPLETED`

**Step 3 — Fetch result (only when COMPLETED):**

```bash
curl -s "https://queue.fal.run/{model-id}/requests/{request_id}/response" \
  -H "Authorization: Key $FAL_KEY" | jq '.'
```

## Synchronous (Fast Models)

```bash
curl -s -X POST "https://fal.run/{model-id}" \
  -H "Authorization: Key $FAL_KEY" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "..."}' | jq '.'
```

## Webhooks

Add `?fal_webhook=https://your-server.com/webhook` to the queue submit URL. fal POSTs when complete:

```json
{ "request_id": "...", "status": "OK", "payload": { ... } }
```

`status` is `"OK"` or `"ERROR"`. Retries up to 10 times over 2 hours — make handlers idempotent.

## Key Differences from Replicate

| | fal.ai | Replicate |
|---|---|---|
| Auth header | `Authorization: Key $FAL_KEY` | `Authorization: Bearer $REPLICATE_API_TOKEN` |
| Model ID | `fal-ai/flux/dev` | `black-forest-labs/flux-dev` |
| Queue URL | `queue.fal.run/{model-id}` | `api.replicate.com/v1/predictions` |
| Result | Separate GET `/response` endpoint | Inline in prediction once `status: succeeded` |
| Webhook param | `?fal_webhook=...` in URL | `webhook` field in POST body |
| Auto-retry | Yes, up to 10x server-side | No |

# Seedance 2.0 Prompt Writer

## Overview

Generates optimized prompts for Seedance 2.0 video generation, supporting:

- Image-to-Video
- Text-to-Video
- Multi-shot scene sequences

Seedance 2.0 responds strongly to explicit motion descriptions, sequential actions, and camera movements. Prompts should prioritize movement and timing over static description.

---

# Input Schema

    {
      "type": "object",
      "properties": {
        "mode": { "type": "string", "enum": ["image_to_video", "text_to_video"] },
        "subject": { "type": "string" },
        "scene": { "type": "string" },
        "actions": { "type": "array", "items": { "type": "string" } },
        "camera": { "type": "array", "items": { "type": "string" } },
        "style": { "type": "string" },
        "intensity": { "type": "string" },
        "shots": {
          "type": "array",
          "description": "For multi-shot sequences",
          "items": {
            "type": "object",
            "properties": {
              "subject": { "type": "string" },
              "action": { "type": "string" },
              "camera": { "type": "string" }
            }
          }
        }
      },
      "required": ["subject"]
    }

---

# Prompt Construction Rules

## Image-to-Video

    [Subject] + [Movement], [Background] + [Movement], [Camera] + [Movement]

Focus on moving parts. Do not describe static elements.

## Text-to-Video

    [Subject] + [Movement] + [Scene] + [Camera] + [Style]

Minimum:

    subject + motion + scene

---

# Writing Principles

## 1. Motion First

Describe subject motion, environment motion, camera motion — in that priority order.

Good:

    A dancer spins rapidly across the stage, spotlights sweeping, camera pulls back to reveal the audience.

## 2. Sequential Action

    subject + action1 → action2 → action3

Example:

    A singer steps to the mic, tilts head back, begins to perform.

## 3. Multi-Character

Chain subjects:

    A guitarist plays fast riffs, drummer hits cymbals hard, vocalist leans into the mic.

## 4. Camera Language

Supported motions:

    pan · zoom · aerial · follow · surround · handheld · push in · tilt · shot switch · pull back · crane

Example:

    Camera slowly pushes in, then shot switch to aerial view of the crowd.

## 5. Intensity Adverbs

Always make intensity explicit:

    fast · violent · large · high frequency · strong · crazy · wildly · rapidly · slowly · gently

---

# Multi-Shot Scene Support

Seedance 2.0 supports shot transitions within a single prompt. Use `Shot switch.` as the delimiter.

## Format

    [Shot 1: subject + motion + camera]

    Shot switch.

    [Shot 2: new subject or angle + motion + camera]

    Shot switch.

    [Shot 3: ...]

## Example — 3-Shot Music Video Scene

    A vocalist stands under a single spotlight, head bowed, then slowly raises arms wide. Camera starts close on hands, slowly pulling back.

    Shot switch.

    Wide aerial shot of the empty arena, single spotlight visible below, audience beginning to fill in from the sides.

    Shot switch.

    Close-up of the vocalist's face, eyes opening slowly, camera holds steady.

---

# Image-to-Video Guidelines

- Follow the image composition — do not contradict visual elements
- Extend the image, do not change it

| Image Content | Bad Prompt | Good Prompt |
|---|---|---|
| man in trench coat | woman dancing | man slowly turns, coat billowing in wind |
| empty rooftop | busy street | camera drifts across the empty rooftop, clouds moving fast overhead |

---

# Best Practices

- Lead with the dominant subject and its motion
- Combine motion layers: subject + environment + camera
- Add emotional dynamics: "nervously", "triumphantly", "with hesitation"
- Keep each shot description self-contained when using multi-shot format

# Failure Cases

Avoid:
- Static description ("A beautiful landscape")
- Negative prompts
- Contradicting the source image
- Vague actions ("something happens")

---

# Example Outputs

## Multi-Shot — Image-to-Video

    Shot switch.

    Wide shot of the rooftop, figure small against the city skyline, clouds moving fast overhead.

    Shot switch.

    Close-up of hands on guitar strings, fingers moving rapidly, shallow depth of field.

Generates optimized prompts for Google Veo 3.1 video generation, supporting:

- Text-to-Video
- Reference-to-Video (image-to-video)

Veo 3.1 excels at cinematic realism, physical accuracy, and naturalistic camera motion. It also generates synchronized audio — prompts can and should include audio environment cues.

## Text-to-Video

    [Subject] + [Action] + [Scene/Environment] + [Camera] + [Audio Cues] + [Style/Mood]

## Reference-to-Video (Image-to-Video)

    [What changes in the image] + [Camera motion] + [Audio cues]

Do not contradict or replace elements in the reference image — only animate and extend.

---

# Writing Principles

## 1. Cinematic Language

Veo 3.1 understands professional film terminology. Use it:

    dolly in · dolly out · tracking shot · crane shot · handheld · steadicam ·
    rack focus · shallow DOF · wide establishing · close-up · medium shot ·
    over-the-shoulder · POV

Example:

    A musician performs at a rain-soaked outdoor stage. Dolly in from wide establishing to medium close-up, rain streaking through the light beams.

## 2. Physical Realism

Veo 3.1 simulates physics well. Describe physical effects explicitly:

    rain hitting puddles · fabric billowing · dust rising from footsteps ·
    hair moving in wind · light refracting through glass · smoke drifting

Example:

    A vinyl record spins on a turntable, dust particles visible in the shaft of light, camera slowly racks focus from the label to the needle.

## 3. Native Audio Cues

Veo 3.1 generates audio alongside video. Include audio cues in prompts:

    "crowd murmuring", "rain on windows", "guitar feedback", "footsteps on gravel",
    "distant sirens", "wind through trees", "applause fading in"

Example:

    A performer bows at the end of a set, crowd applause rising, camera pulls back to reveal the full venue.

## 4. Sequential Action

    subject + action1, then action2, camera responding

Example:

    A dancer leaps from a platform, lands with a heavy thud echoing in the hall, camera follows the fall from above then snaps to a low angle on impact.

## 5. Multi-Character

    Character A does X. Character B responds. Camera mediates.

Example:

    Two guitarists face each other, trading riffs, the camera cuts between close-ups of their hands and a wide two-shot.

---

# Multi-Shot Scene Support

Veo 3.1 supports scene transitions. Use `Cut to:` or `Scene transition:` as delimiters.

## Format

    [Shot 1: subject + action + camera + audio]

    Cut to:

    [Shot 2: new subject or angle + action + camera + audio]

## Example — 3-Shot Concert Sequence

    A vocalist stands at the front of a dark stage, a single spotlight cutting through haze. They raise the microphone slowly. Crowd noise swells. Steady camera, slow dolly in.

    Cut to:

    Aerial shot of the arena floor from above, thousands of phone lights appearing one by one as the singer begins. Ambient crowd noise and first notes of the song.

    Cut to:

    Close-up of the vocalist's closed eyes, face lit by the spotlight, sweat visible. Camera holds still. Room tone quiet beneath the vocals.

---

# Reference-to-Video Guidelines

When using a reference image:

- Describe what changes — motion, lighting shifts, atmospheric effects
- Do not contradict visual content (hair color, scene type, character gender)
- Extend the image naturally

| Image Content | Bad Prompt | Good Prompt |
|---|---|---|
| woman at window | man walks in | she slowly turns toward the camera, rain streaking the glass behind her |
| empty street at dusk | busy marketplace | camera drifts down the empty street, a distant siren fading in |

---

# Best Practices

- Describe the complete visual environment, not just the subject
- Include lighting conditions: "harsh back-lighting", "golden hour", "neon-lit", "silhouette"
- Match prompt tone to desired mood: terse prompts → raw/handheld, elaborate prompts → cinematic
- Audio and visual descriptions can work together to set atmosphere

# Failure Cases

Avoid:
- Vague actions ("something dramatic happens")
- Contradicting the reference image
- Purely static descriptions with no motion or camera movement
- Overloading a single prompt with too many simultaneous actions

---

## Multi-Shot — Reference-to-Video

    The figure in the doorway slowly steps forward into the light, head tilting up. Shallow depth of field, camera holds on face. Quiet ambient room tone.

    Cut to:

    Wide shot from outside the building looking in through the window, the figure now visible as a silhouette. Camera holds still. Rain begins to streak the glass.

    Cut to:

    Extreme close-up of fingers pressing piano keys. Camera slowly racks focus from hands to the reflection in the piano's surface. Sound of the first chord resonating.
